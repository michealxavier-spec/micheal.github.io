ES6 Introduction

ES6 (ECMAScript 2015) is a major evolution of JavaScript that introduced modern syntax, improved readability, and solved long-standing issues in variable scope, functions, and modular programming.

Before ES6, JavaScript was less structured and often led to unpredictable behavior in large applications. ES6 introduced a cleaner and scalable approach that is now the foundation of modern frameworks like React, Angular, and Node.js.

Why developers prefer ES6

Because it:

  • reduces bugs
  • improves readability
  • supports modern frameworks like React, Angular, and Vue.js
  • makes large applications manageable
  • provides modern programming features similar to Python/Java/C#
improves readability supports modern frameworks like: React Angular Vue.js makes large applications manageable provides modern programming features similar to Python/Java/C#

1. var vs let vs const

This feature introduces proper variable scoping in JavaScript. It is one of the most critical improvements in ES6 because it directly affects how variables behave inside blocks, loops, and functions.

Why we use it?

Before ES6, developers faced unpredictable behavior due to var. ES6 fixed this by introducing:

  • Block-level scope control
  • Prevention of accidental redeclaration
  • Safer and more predictable code execution

Before ES6 (var behavior problem)

var x = 100;

if(true){
  var x = 200;
}

console.log(x);
Output: 200

👉 Problem: var ignores block scope and overrides outer value.

ES6 Solution (let & const)

let x = 100;

if(true){
  let x = 200;
  console.log("Inside block:", x);
}

console.log("Outside block:", x);
Output:
Inside block: 200
Outside block: 100

const behavior example

const API_URL = "https://api.example.com";

console.log(API_URL);
Output: https://api.example.com

Key Differences (Important)

  • var → function scoped, can redeclare, unsafe in large apps
  • let → block scoped, can reassign, safer
  • const → block scoped, cannot reassign (recommended default) But we can use it for objects and arrays to modify their properties

Summary

ES6 introduced let and const to eliminate scope-related bugs. In modern JavaScript, var is avoided completely. Always prefer const by default and use let only when reassignment is needed.

2. Arrow Functions

Arrow functions are a modern ES6 syntax for writing functions in a shorter and cleaner way. They also behave differently from normal functions in how they handle this keyword.

Why we use Arrow Functions?

  • Shorter syntax compared to normal functions
  • Improves readability in callbacks
  • No own this binding (important behavior)
  • Commonly used in React, Promises, Array methods

Before ES6 (Normal Function)

function add(a, b) {
  return a + b;
}

console.log(add(5, 3));
Output: 8

ES6 Arrow Function

const add = (a, b) => {
  return a + b;
};

console.log(add(5, 3));
Output: 8

Short Syntax (Implicit Return)

const multiply = (a, b) => a * b;

console.log(multiply(4, 2));
Output: 8

Arrow Function with Single Parameter

const square = x => x * x;

console.log(square(6));
Output: 36

Important Concept: this keyword difference

Arrow functions do NOT have their own this. They inherit this from the parent scope.

Normal Function (this behavior)

const obj = {
  name: "John",
  show: function () {
    console.log(this.name);
  }
};

obj.show();
Output: John

Arrow Function (this behavior)

const obj = {
  name: "John",
  show: () => {
    console.log(this.name);
  }
};

obj.show();
Output: undefined

👉 Because arrow function does not have its own this

Real World Example (setTimeout issue fix)

function Person() {
  this.name = "Alex";

  setTimeout(() => {
    console.log(this.name);
  }, 1000);
}

new Person();
Output: Alex

👉 Arrow function fixes this binding problem in async callbacks.

Summary

Arrow functions provide shorter syntax and solve common this binding issues. However, they should NOT be used when you need dynamic this (like object methods or constructors).

3. Template Literals

Template literals (introduced in ES6) provide a cleaner and more powerful way to work with strings using backticks (`) instead of quotes. They support variables, expressions, and multi-line strings.

Why we use Template Literals?

  • Easy string interpolation (embed variables inside strings)
  • Supports multi-line strings without concatenation
  • Makes code more readable and maintainable
  • Used heavily in API responses, UI rendering, and logging

Before ES6 (String Concatenation Problem)

var name = "John";
var age = 25;

var message = "My name is " + name + " and I am " + age + " years old.";

console.log(message);
Output: My name is John and I am 25 years old.

ES6 Template Literals

const name = "John";
const age = 25;

const message = `My name is ${name} and I am ${age} years old.`;

console.log(message);
Output: My name is John and I am 25 years old.

Expression inside Template Literals

const a = 10;
const b = 20;

console.log(`Sum of a and b is ${a + b}`);
Output: Sum of a and b is 30

Multi-line String (Before ES6)

var text = "Line 1\n" +
           "Line 2\n" +
           "Line 3";

console.log(text);
Output:
Line 1
Line 2
Line 3

Multi-line String (ES6)

const text = `
Line 1
Line 2
Line 3
`;

console.log(text);
Output:
Line 1
Line 2
Line 3

Real-world Usage Example (API / UI)

const user = {
  name: "Alex",
  role: "Developer"
};

const card = `
  User Info:
  Name: ${user.name}
  Role: ${user.role}
`;

console.log(card);
Output:
User Info:
Name: Alex
Role: Developer

Summary

Template literals simplify string handling in JavaScript. They eliminate complex concatenation, support expressions, and make code much more readable—especially in dynamic UI rendering and API-based applications.

4. Destructuring (Objects & Arrays)

Destructuring is an ES6 feature that allows you to extract values from arrays or objects into separate variables in a clean and readable way. It reduces repetitive code and improves clarity.

Why we use Destructuring?

  • Reduces repetitive object.property access
  • Makes code cleaner and shorter
  • Very useful in API responses and React props
  • Improves readability in large applications

Before ES6 (Object Access)

const user = {
  name: "John",
  age: 25,
  city: "London"
};

console.log(user.name);
console.log(user.age);
console.log(user.city);
Output:
John
25
London

ES6 Object Destructuring

const user = {
  name: "John",
  age: 25,
  city: "London"
};

const { name, age, city } = user;

console.log(name);
console.log(age);
console.log(city);
Output:
John
25
London

Renaming Variables

const user = {
  name: "John",
  age: 25
};

const { name: userName, age: userAge } = user;

console.log(userName);
console.log(userAge);
Output:
John
25

Default Values

const user = {
  name: "Alex"
};

const { name, age = 30 } = user;

console.log(name);
console.log(age);
Output:
Alex
30

Array Destructuring

const colors = ["red", "green", "blue"];

const [first, second, third] = colors;

console.log(first);
console.log(second);
console.log(third);
Output:
red
green
blue

Skipping Values in Array

const numbers = [10, 20, 30, 40];

const [a, , c] = numbers;

console.log(a);
console.log(c);
Output:
10
30

Real-world API Example

const response = {
  status: 200,
  data: {
    user: "John",
    role: "Admin"
  }
};

const {
  data: { user, role }
} = response;

console.log(user);
console.log(role);
Output:
John
Admin

Summary

Destructuring is a powerful ES6 feature that simplifies extracting values from objects and arrays. It reduces boilerplate code and is widely used in modern frameworks like React and Node.js for handling props and API responses.

5. Spread & Rest Operators (...)

The ... operator in ES6 is used for both Spread and Rest operations. Even though the syntax looks the same, their usage depends on the context.

Why we use Spread & Rest?

  • To copy or merge arrays and objects easily (Spread)
  • To collect multiple values into a single variable (Rest)
  • Used heavily in React state updates and function arguments

1. Spread Operator (Arrays)

Spread is used to expand elements of an array.

Before ES6 (manual merging)

const arr1 = [1, 2];
const arr2 = [3, 4];

const result = arr1.concat(arr2);

console.log(result);
Output: [1, 2, 3, 4]

ES6 (Spread Operator)

const arr1 = [1, 2];
const arr2 = [3, 4];

const result = [...arr1, ...arr2];

console.log(result);
Output: [1, 2, 3, 4]

2. Spread Operator (Objects)

const user = {
  name: "John",
  age: 25
};

const updatedUser = {
  ...user,
  city: "London"
};

console.log(updatedUser);
Output: { name: "John", age: 25, city: "London" }

3. Copying Arrays (Important Use Case)

const numbers = [10, 20, 30];

const copy = [...numbers];

console.log(copy);
Output: [10, 20, 30]

4. Rest Operator (Function Arguments)

Rest operator collects multiple values into a single array.

Before ES6

function sum() {
  console.log(arguments);
}

sum(1, 2, 3);
Output: Arguments object (not a real array)

ES6 Rest Operator

function sum(...numbers) {
  console.log(numbers);
}

sum(1, 2, 3, 4);
Output: [1, 2, 3, 4]

5. Rest with Parameters

function total(a, b, ...rest) {
  console.log(a);
  console.log(b);
  console.log(rest);
}

total(10, 20, 30, 40, 50);
Output:
10
20
[30, 40, 50]

Key Difference

  • Spread → Expands values
  • Rest → Collects values

Summary

Spread and Rest operators simplify working with arrays, objects, and function parameters. Spread is used for copying and merging, while Rest is used for collecting multiple values into a single structure.

6. ES6 Modules (import / export)

ES6 Modules allow us to split JavaScript code into multiple files and reuse code easily using export and import. This makes applications more scalable and maintainable.

Why we use Modules?

  • Break large code into smaller files
  • Improve code reusability
  • Better maintainability
  • Used in React, Node.js, modern frameworks

Before ES6 (Problem)

Before modules, all code was written in one file or global scope, which caused:
  • Variable conflicts
  • Difficult debugging
  • Hard to maintain large applications
function add(a, b) {
  return a + b;
}

function multiply(a, b) {
  return a * b;
}

console.log(add(2, 3));
console.log(multiply(2, 3));
Output:
5
6

ES6 Solution (Modules)

Now we can split code into multiple files using export and import.

1. Named Export

// math.js

export function add(a, b) {
  return a + b;
}

export function multiply(a, b) {
  return a * b;
}

2. Importing Functions

// main.js

import { add, multiply } from "./math.js";

console.log(add(2, 3));
console.log(multiply(2, 3));
Output:
5
6

3. Default Export

// user.js

export default function greet(name) {
  return `Hello ${name}`;
}

Import Default Export

// main.js

import greet from "./user.js";

console.log(greet("John"));
Output: Hello John

4. Mixed Export Example

// utils.js

export const PI = 3.14;

export default function square(x) {
  return x * x;
}

// main.js

import square, { PI } from "./utils.js";

console.log(PI);
console.log(square(4));
Output:
3.14
16

Important Notes

  • Modules work only in modern browsers or Node.js
  • Use type="module" in HTML script tag
  • Each file is treated as a separate module

Summary

ES6 Modules allow developers to split code into reusable pieces using import and export. This improves scalability, readability, and maintainability of modern JavaScript applications.

7. Closures in JavaScript

A Closure is created when a function remembers variables from its outer scope even after the outer function has finished execution.

Why we use Closures?

  • To preserve data between function calls
  • To create private variables (data hiding)
  • Used in callbacks, event handlers, and React hooks
  • Helps in building secure and modular code

Before Understanding Closures (Problem)

function outer() {
  let count = 0;
  count++;
  console.log(count);
}

outer();
outer();
outer();
Output:
1
1
1

👉 Problem: value resets every time function runs.

ES6 Concept: Closure Solution

function outer() {
  let count = 0;

  return function () {
    count++;
    console.log(count);
  };
}

const counter = outer();

counter();
counter();
counter();
Output:
1
2
3

How Closure Works (Concept)

The inner function "remembers" the variables of the outer function even after execution is completed.
  • outer() runs once
  • returns inner function
  • inner function still has access to count

Real-world Example: Private Variable

function bankAccount() {
  let balance = 1000;

  return {
    deposit(amount) {
      balance += amount;
      console.log("Balance:", balance);
    },
    withdraw(amount) {
      balance -= amount;
      console.log("Balance:", balance);
    }
  };
}

const account = bankAccount();

account.deposit(500);
account.withdraw(200);
Output:
Balance: 1500
Balance: 1300

👉 balance is private and cannot be accessed directly.

Real-world Usage

  • Data encapsulation
  • React hooks (useState behavior internally)
  • Event listeners storing state
  • Function factories

Summary

Closures allow a function to remember and access variables from its outer scope even after execution. They are powerful for data privacy, state management, and advanced JavaScript patterns.

8. this Keyword in JavaScript

The this keyword refers to the object it belongs to. However, its value depends on how and where the function is called. This makes it one of the most confusing topics in JavaScript.

Why we use "this"?

  • To access object properties inside methods
  • To refer to current execution context
  • Used in constructors and classes
  • Important in event handling and callbacks

1. this in Global Scope

console.log(this);
Output: Window (Browser) / Global object (Node.js)

👉 In global scope, this refers to the global object.

2. this inside Object Method

const user = {
  name: "John",
  show: function () {
    console.log(this.name);
  }
};

user.show();
Output: John

👉 here this refers to the object itself (user)

3. this in Normal Function

function test() {
  console.log(this);
}

test();
Output: Window / undefined (strict mode)

👉 In normal functions, this depends on strict mode.

4. this Problem in Nested Function

const user = {
  name: "John",
  show: function () {
    function inner() {
      console.log(this.name);
    }
    inner();
  }
};

user.show();
Output: undefined

👉 inner function loses reference of object context.

5. Solution using Arrow Function

const user = {
  name: "John",
  show: function () {
    const inner = () => {
      console.log(this.name);
    };
    inner();
  }
};

user.show();
Output: John

👉 Arrow function inherits this from parent scope.

6. this with setTimeout Problem

const user = {
  name: "Alex",
  show: function () {
    setTimeout(function () {
      console.log(this.name);
    }, 1000);
  }
};

user.show();
Output: undefined

7. Fix using Arrow Function

const user = {
  name: "Alex",
  show: function () {
    setTimeout(() => {
      console.log(this.name);
    }, 1000);
  }
};

user.show();
Output: Alex

Summary

The value of "this" depends on execution context. Arrow functions do not have their own "this" and inherit it from parent scope, which helps solve many common JavaScript bugs.

9. call / apply / bind in JavaScript

These three methods are used to control the value of this in functions. They allow us to borrow functions from one object and use them in another context.

Why we use call / apply / bind?

  • To explicitly set the value of this
  • To reuse functions across objects
  • To pass arguments in controlled ways
  • Used in event handling and functional programming

1. call() Method

call() executes the function immediately with a given this value. 
const user = {
  name: "John"
};

function greet() {
  console.log("Hello " + this.name);
}

greet.call(user);
Output: Hello John

call() with arguments

function intro(age) {
  console.log(this.name + " is " + age + " years old");
}

const person = { name: "Alex" };

intro.call(person, 25);
Output: Alex is 25 years old

2. apply() Method

apply() is similar to call(), but arguments are passed as an array. 
function intro(age, city) {
  console.log(this.name + " is " + age + " from " + city);
}

const person = { name: "Sam" };

intro.apply(person, [30, "London"]);
Output: Sam is 30 from London

call vs apply difference

  • call() → arguments passed individually
  • apply() → arguments passed as array

3. bind() Method

bind() does NOT execute immediately. It returns a new function with bound this value. 
const user = {
  name: "Emma"
};

function greet() {
  console.log("Hello " + this.name);
}

const boundFunc = greet.bind(user);

boundFunc();
Output: Hello Emma

bind() with arguments

function intro(age) {
  console.log(this.name + " is " + age);
}

const person = { name: "David" };

const bound = intro.bind(person, 28);

bound();
Output: David is 28

Real-world Usage Example

const button = {
  text: "Click Me",
  click: function () {
    console.log(this.text);
  }
};

const btnClick = button.click.bind(button);

btnClick();
Output: Click Me

Summary

call and apply execute functions immediately with controlled this, while bind returns a new function. These methods are essential for controlling execution context in JavaScript.

10. Pure Functions & Immutability

These are core concepts of functional programming in JavaScript. They help in writing predictable, bug-free, and maintainable code.

Why we use them?

  • To avoid unexpected side effects
  • To make code predictable and testable
  • Used heavily in React and Redux
  • Improves performance and debugging

1. Pure Functions

A pure function always returns the same output for the same input and does not modify external state.

Example of Pure Function

function add(a, b) {
  return a + b;
}

console.log(add(2, 3));
console.log(add(2, 3));
Output:
5
5

Impure Function Example

let total = 0;

function addToTotal(value) {
  total += value;
  return total;
}

console.log(addToTotal(5));
console.log(addToTotal(5));
Output:
5
10

👉 Problem: Output depends on external state (not predictable)

2. Immutability

Immutability means not changing original data. Instead, we create a new copy with modifications.

Before ES6 (Mutating data)

const user = {
  name: "John",
  age: 25
};

user.age = 30;

console.log(user);
Output: { name: "John", age: 30 }

ES6 Way (Immutability)

const user = {
  name: "John",
  age: 25
};

const updatedUser = {
  ...user,
  age: 30
};

console.log(updatedUser);
Output: { name: "John", age: 30 }

Immutability with Arrays

const numbers = [1, 2, 3];

const newNumbers = [...numbers, 4];

console.log(newNumbers);
Output: [1, 2, 3, 4]

Real-world Usage

  • React state updates
  • Redux reducers
  • Functional programming patterns
  • Predictable data flow systems

Summary

Pure functions ensure predictable behavior without side effects, while immutability prevents direct data modification. Together, they form the foundation of modern JavaScript architecture.

11. Promises in JavaScript

A Promise in JavaScript is an object that represents the eventual completion (or failure) of an asynchronous operation. It is used to handle async tasks like API calls, file loading, and database operations.

Why we use Promises?

  • To handle asynchronous operations cleanly
  • To avoid callback hell
  • To make code more readable and structured
  • Used in API calls (fetch, axios, etc.)

Before Promises (Callback Hell Problem)

setTimeout(() => {
  console.log("Step 1");

  setTimeout(() => {
    console.log("Step 2");

    setTimeout(() => {
      console.log("Step 3");
    }, 1000);

  }, 1000);

}, 1000);
Output:
Step 1
Step 2
Step 3

👉 Problem: deeply nested callbacks become hard to maintain.

1. Creating a Promise

const myPromise = new Promise((resolve, reject) => {

  let success = true;

  if (success) {
    resolve("Operation Successful");
  } else {
    reject("Operation Failed");
  }

});

console.log(myPromise);
Output: Promise { "Operation Successful" }

2. Handling Promise (then / catch)

myPromise
  .then((result) => {
    console.log(result);
  })
  .catch((error) => {
    console.log(error);
  });
Output: Operation Successful

3. Promise States

  • Pending → Initial state
  • Fulfilled → Operation successful
  • Rejected → Operation failed

4. Real-world Example (API Simulation)

function fetchData() {
  return new Promise((resolve, reject) => {

    setTimeout(() => {
      resolve("Data fetched from server");
    }, 2000);

  });
}

fetchData()
  .then((data) => {
    console.log(data);
  });
Output (after 2 sec): Data fetched from server

5. Promise Chaining

function step1() {
  return new Promise(resolve => {
    resolve("Step 1 completed");
  });
}

function step2() {
  return new Promise(resolve => {
    resolve("Step 2 completed");
  });
}

step1()
  .then((res) => {
    console.log(res);
    return step2();
  })
  .then((res) => {
    console.log(res);
  });
Output:
Step 1 completed
Step 2 completed

Promise Combinators Summary

  • Promise.all() → All must succeed
  • Promise.allSettled() → Wait for all results
  • Promise.race() → First completed wins
  • Promise.any() → First successful wins

Summary

Promises are used to handle asynchronous operations in a structured way. They replace callback hell with cleaner chaining using then and catch, making code more readable and maintainable.

12. Async / Await in JavaScript

Async/Await is a modern way to handle asynchronous operations in JavaScript. It is built on top of Promises and makes async code look like synchronous code.

Why we use Async / Await?

  • Makes code more readable and cleaner
  • Removes .then() chaining complexity
  • Easier error handling using try/catch
  • Used in API calls and backend operations

Before Async/Await (Promise Style)

function fetchData() {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve("Data received");
    }, 2000);
  });
}

fetchData().then((data) => {
  console.log(data);
});
Output (after 2 sec): Data received

ES6+ Async / Await Version

function fetchData() {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve("Data received");
    }, 2000);
  });
}

async function getData() {
  const result = await fetchData();
  console.log(result);
}

getData();
Output (after 2 sec): Data received

How Async / Await Works

  • async → makes a function return a Promise
  • await → pauses execution until Promise is resolved

Async Function Example

async function example() {
  return "Hello World";
}

example().then((res) => {
  console.log(res);
});
Output: Hello World

Error Handling (try / catch)

async function fetchData() {
  return new Promise((resolve, reject) => {
    reject("Error occurred!");
  });
}

async function getData() {
  try {
    const result = await fetchData();
    console.log(result);
  } catch (error) {
    console.log(error);
  }
}

getData();
Output: Error occurred!

Multiple Await Example

function step1() {
  return new Promise(resolve => {
    setTimeout(() => resolve("Step 1 done"), 1000);
  });
}

function step2() {
  return new Promise(resolve => {
    setTimeout(() => resolve("Step 2 done"), 1000);
  });
}

async function runSteps() {
  const a = await step1();
  console.log(a);

  const b = await step2();
  console.log(b);
}

runSteps();
Output:
Step 1 done
Step 2 done

Summary

Async/Await simplifies asynchronous programming by making code look synchronous. It improves readability, reduces complexity, and is now the standard for handling async operations in modern JavaScript.

13. Execution Context & Call Stack

Execution Context is the environment where JavaScript code is executed. Call Stack is a structure that manages the execution of functions in a Last In First Out (LIFO) order.

Why we need Execution Context?

  • To understand how JavaScript runs code internally
  • To manage function execution order
  • To handle memory creation and execution phases

1. Execution Context Phases

  • Creation Phase → Memory is allocated for variables and functions
  • Execution Phase → Code is executed line by line

Example Code

var a = 10;

function add(b) {
  var result = a + b;
  return result;
}

var sum = add(5);

console.log(sum);

Step-by-step Execution

1. Global Execution Context is created
2. Variables (a, add, sum) are stored in memory phase
3. Execution starts line by line
4. add(5) creates a new function execution context
5. Result is calculated and returned

2. Call Stack Concept

Call Stack is like a stack of plates. The last function pushed is the first one executed.

Call Stack Flow Example

function first() {
  console.log("First");
  second();
}

function second() {
  console.log("Second");
  third();
}

function third() {
  console.log("Third");
}

first();
Output:
First
Second
Third

Call Stack Flow Explanation

  • first() → pushed into stack
  • second() → pushed inside first()
  • third() → pushed inside second()
  • then execution unwinds in reverse order

3. Stack Visualization Idea

Call Stack works like:

first() → second() → third()
then:
third() → second() → first()

4. Stack Overflow Example

function test() {
  test();
}

test();
Output: Maximum call stack size exceeded

👉 Infinite recursion fills the call stack and causes crash.

Summary

Execution Context defines how JavaScript code is executed in memory and runtime phases. Call Stack manages function execution order using LIFO structure. Understanding this is essential to master JavaScript internals.

14. Event Loop in JavaScript

The Event Loop is a mechanism that allows JavaScript to handle asynchronous operations even though it is single-threaded. It connects the Call Stack, Web APIs, Callback Queue, and Microtask Queue.

Why we need Event Loop?

  • JavaScript is single-threaded (one task at a time)
  • Helps handle asynchronous tasks (API calls, timers)
  • Ensures non-blocking execution

Core Components

  • Call Stack → Executes functions
  • Web APIs → Handles async tasks (setTimeout, fetch)
  • Callback Queue → Stores completed async callbacks
  • Microtask Queue → Stores Promises (higher priority)
  • Event Loop → Moves tasks into Call Stack

Example Code

console.log("Start");

setTimeout(() => {
  console.log("Timeout");
}, 0);

console.log("End");
Output:
Start
End
Timeout

How it works (Step-by-step)

  • "Start" goes to Call Stack → executes immediately
  • setTimeout goes to Web API
  • "End" executes next
  • setTimeout callback moves to Callback Queue
  • Event Loop pushes it to Call Stack when empty

Microtask vs Callback Queue

console.log("Start");

setTimeout(() => {
  console.log("Timeout");
}, 0);

Promise.resolve().then(() => {
  console.log("Promise");
});

console.log("End");
Output:
Start
End
Promise
Timeout

Important Insight

Microtasks (Promises) always execute before Callback Queue (setTimeout), even if delay is 0.

Real-world Example

console.log("User Login");

setTimeout(() => {
  console.log("Fetching user data...");
}, 1000);

console.log("UI Loaded");
Output:
User Login
UI Loaded
Fetching user data...

Summary

The Event Loop enables JavaScript to perform asynchronous operations efficiently. It continuously monitors the Call Stack and queues, ensuring non-blocking execution and proper task ordering.

15. Debouncing

Debouncing is a performance optimization technique used to control how often a function is executed when events fire repeatedly (like scroll, resize, keypress).

Why we use them?

  • To improve performance
  • To reduce unnecessary API calls
  • To prevent function over-execution
  • Used in search bars, scroll events, resizing

1. Debouncing

Debouncing ensures that a function is executed only after a certain delay once the event stops firing.

Example: Search Input

let timer;

function searchData(query) {
  console.log("Searching:", query);
}

function debounceSearch(value) {
  clearTimeout(timer);

  timer = setTimeout(() => {
    searchData(value);
  }, 1000);
}

debounceSearch("H");
debounceSearch("He");
debounceSearch("Hel");
debounceSearch("Hell");
debounceSearch("Hello");
Output (after 1 sec pause): Searching: Hello

👉 Only the final input is processed after user stops typing.

Real-world Usage

  • Search suggestions (Debounce)
  • Button spam prevention (Debounce)

Summary

Debouncing and Throttling are performance optimization techniques used to control function execution rate. Debounce waits for inactivity, while throttle limits execution frequency.

17. Hoisting in JavaScript

Hoisting is JavaScript's default behavior of moving declarations (variables and functions) to the top of their scope before code execution.

Why we use Hoisting concept?

  • To understand unexpected behavior in code execution
  • To debug variable and function access issues
  • To understand execution context better

1. Variable Hoisting (var)

console.log(a);

var a = 10;
Output: undefined

👉 Declaration is hoisted, but initialization is NOT.

How JavaScript interprets it

var a;
console.log(a);
a = 10;

2. let and const Hoisting

console.log(b);

let b = 20;
Output: ReferenceError: Cannot access 'b' before initialization

👉 let and const are hoisted but placed in Temporal Dead Zone (TDZ)

3. Function Hoisting

greet();

function greet() {
  console.log("Hello World");
}
Output: Hello World

👉 Function declarations are fully hoisted.

4. Function Expression Hoisting

sayHello();

var sayHello = function () {
  console.log("Hello");
};
Output: TypeError: sayHello is not a function

👉 Only variable declaration is hoisted, not the function assignment.

5. Hoisting Summary Table

  • var → hoisted (undefined)
  • let / const → hoisted but in TDZ
  • function declaration → fully hoisted
  • function expression → not hoisted as function

Real-world Importance

  • Avoids confusion in variable usage
  • Helps understand runtime errors
  • Critical for debugging JS code

Summary

Hoisting is JavaScript's behavior of moving declarations to the top of their scope before execution. Understanding hoisting helps prevent common bugs and improves understanding of execution context.

16. Memoization in JavaScript

Memoization is an optimization technique used to store the results of expensive function calls and return cached results when the same inputs occur again.

Why we use Memoization?

  • To avoid repeated calculations
  • To improve performance of expensive operations
  • Used in recursion optimization (like Fibonacci)
  • Helps in caching API or computed results

Without Memoization (Slow Execution)

function slowAdd(n) {
  console.log("Calculating...");
  return n + 10;
}

console.log(slowAdd(5));
console.log(slowAdd(5));
console.log(slowAdd(5));
Output:
Calculating... 15
Calculating... 15
Calculating... 15

👉 Problem: Same calculation happens again and again.

With Memoization (Optimized)

function memoizedAdd() {
  let cache = {};

  return function (n) {
    if (cache[n]) {
      console.log("From Cache");
      return cache[n];
    } else {
      console.log("Calculating...");
      let result = n + 10;
      cache[n] = result;
      return result;
    }
  };
}

const add = memoizedAdd();

console.log(add(5));
console.log(add(5));
console.log(add(5));
Output:
Calculating... 15
From Cache 15
From Cache 15

How Memoization Works

  • Stores results in a cache object
  • Checks cache before computing
  • Returns cached result if available
  • Uses closures to persist cache memory

Key Idea

Memoization improves performance by avoiding repeated execution of expensive functions using cached results.

Summary

Memoization is a powerful optimization technique that uses caching to store function results. It significantly improves performance in repetitive or computationally expensive operations.

17. OOP Concepts in JavaScript

Object-Oriented Programming (OOP) is a programming paradigm based on objects and classes. It helps structure code in a modular, reusable, and scalable way. JavaScript supports OOP using prototypes and ES6 classes.

Why we use OOP?

  • To structure large applications
  • To improve code reusability
  • To model real-world entities
  • To make code scalable and maintainable

1. Basic Object Example

const user = {
  name: "John",
  age: 25,
  greet() {
    console.log("Hello " + this.name);
  }
};

user.greet();
Output: Hello John

function Person(name, age) {

  this.age = age;

const p1 = new Person("Alex", 30);

p1.greet();
p2.greet();
Output:
Hi Alex
Hi John

👉 Each object gets its own copy of methods (not memory efficient)

3. ES6 Classes (Modern OOP)

Feature Factory Function Class

Large apps Difficult Organized

4. 4 Pillars of OOP

Encapsulation

Bundling data and methods inside a class. 
class BankAccount {
  constructor(balance) {
    this.balance = balance;
  }

  deposit(amount) {
    this.balance += amount;
  }

  getBalance() {
    return this.balance;
  }
}

const acc = new BankAccount(1000);
acc.deposit(500);

console.log(acc.getBalance());
Output: 1500 
class BankAccount {
  #balance;

  constructor(balance) {
    this.#balance = balance;
  }

  deposit(amount) {
    this.#balance += amount;
  }

  getBalance() {
    return this.#balance;
  }
}

const acc = new BankAccount(1000);
Output: acc.#balance = 999999; // Error

Inheritance

One class can inherit properties and methods from another class. 
class Animal {
  speak() {
    console.log("Animal speaks");
  }
}

  bark() {
    console.log("Dog barks");
  }

Animal speaks
Dog barks

super Keyword

class Parent {
  constructor(name) {
    this.name = name;
  }
}

class Child extends Parent {
  constructor(name, age) {
    super(name);
    this.age = age;
  }
}

Hiding internal implementation and showing only required functionality.

Engine Started

Summary

OOP in JavaScript helps structure code using objects and classes. It is based on 4 pillars: Encapsulation, Abstraction, Inheritance, and Polymorphism. ES6 classes provide a cleaner syntax over prototypes.
